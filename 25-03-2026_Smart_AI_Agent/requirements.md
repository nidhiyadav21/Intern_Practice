### Functional Requirements
* The system must allow users to create an account to become a customer.
* The system must allow customers to browse, search, select, and add books to a shopping cart.
* The system must allow customers to check out books in their shopping cart and decrement the stock that the inventory the system maintains.
* The system must allow managers to manage the inventory with full create, retrieve, update, and delete (CRUD) functionality with regards to books in the system.
* The system must allow customers and managers to interact with a promotion system that handles percentage-off promotions that can be applied to member's orders.
* The system must have full email capabilities to send promotions to members of the system and provide managers with low-stock notifications.
* The system must allow managers to specify a stop-order for a book.
* The system must notify managers when books need to be reordered.
* The system must allow managers to update stock quantities and change any book's price.
* The system must allow managers to view transaction logs.
* The system must allow managers to create promotions.

### Non-Functional Requirements
* The system must be available to anyone using the Computer Science Department's provided computer resources in the MSU Engineering Building.
* The system must work correctly in both Internet Explorer and Mozilla Firefox.
* The system must not have full credit-card processing capabilities.
* The system must not allow managers to be customers.
* The system must not allow multiple promotions to be added to a single shopping cart.
* The system must not allow customers to add more than one of each item to their cart.
* The system must not allow users to retrieve passwords or edit their user details.
* Security is not a concern for this system, and the database may store passwords in plain text.
* The system may not work correctly when used with Internet browsers other than Firefox and Internet Explorer.
